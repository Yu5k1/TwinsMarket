# 限价单提交 → 撮合完整路径分析

## 结论：代码逻辑正确，无 bug

限价单"立刻成交"是**标准交易所行为**，不是代码缺陷。限价单的定义是"以**指定价格或更优价格**成交"，如果挂单价已经跨越买卖价差，就必须立即撮合。

---

## 完整代码路径

```
前端 index.html:1573 → 用户输入 price
    ↓
api.py POST /api/order (line 61-65)
    Order(order_id=..., owner='user', side='buy', type='limit', price=用户输入价, size=...)
    ↓

    ↓
engine.py _match_limit() (line 218-229)
    ├─ 买盘: order.price >= best_ask  →  YES → _match_market()  ← 立刻成交
    │                                    NO  → _insert_order()  ← 挂入买单簿
    └─ 卖盘: order.price <= best_bid  →  YES → _match_market()  ← 立刻成交
                                         NO  → _insert_order()  ← 挂入卖单簿
    ↓
engine.py _insert_order() (line 231-246)
    放入 bids 或 asks SortedDict，不做撮合
```

## `_match_limit` 关键代码 (engine.py:218-229)

```python
async def _match_limit(self, order: Order) -> list[Trade]:
    if order.side == 'buy':
        best_ask = next(iter(self.asks), None)
        if best_ask and order.price >= best_ask:   # 限价 ≥ 最低卖价
            return await self._match_market(order)  # → 立即撮合
    else:
        best_bid = next(iter(self.bids), None)
        if best_bid and order.price <= best_bid:   # 限价 ≤ 最高买价
            return await self._match_market(order)  # → 立即撮合

    self._insert_order(order)  # 不跨价差 → 挂入订单簿
    return []
```

## 两项检查的结论

| 检查项 | 结论 |
|--------|------|
| 限价买单是否错误地直接撮合? | **否**。`order.price >= best_ask` 才会撮合，这是正确的跨价差判断 |
| 限价卖单是否错误地直接撮合? | **否**。`order.price <= best_bid` 才会撮合，正确 |
| 插入订单簿后是否触发撮合? | **否**。`_insert_order()` 只做 `book[order.price].append(order)` + `_update_mid()`，`_update_mid()` 只算中间价，不触发撮合 |

## 为什么会"立刻成交"

当前买卖价差很小（aggressive MM base_spread = 0.04%，conservative = 0.12%），以 mid ≈ 16000 为例：

| 场景 | best_ask | 用户挂买单 | 结果 |
|------|----------|-----------|------|
| A | 16003 | 16000 | **挂入簿中**（16000 < 16003，不跨价差） |
| B | 16003 | 16005 | **立刻成交**（16005 ≥ 16003，跨价差） |

B 场景中，用户出价 16005 买入，市场上有人 16003 卖出——引擎以 16003 撮合，用户拿到了比心理价位**更优**的价格。这就是限价单的"or better"语义。

## 如果想要"等价格到了再触发"

那需要的是**止损单（Stop Order）**，不是限价单：

| 订单类型 | 含义 | 触发时机 |
|----------|------|---------|
| 限价买单 (Limit Buy) | 以 ≤X 价格买入 | 有卖价 ≤X 时立即撮合 |
| 止损买单 (Stop Buy) | 价格涨到 X 时买入 | 价格 ≥X 时才转为市价单 |

后端已支持 `type='stop'`（engine.py:141-142），但前端**没有止损单的 UI**。
